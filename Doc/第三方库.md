## zxing-cpp

如果只编译核心库，不编译示例，可以直接执行：

```cmake
cmake -S zxing-cpp -B zxing-cpp.build -G "Visual Studio 16 2019" -A x64
cmake --build zxing-cpp.release -j8 --config Debug
```

如果要编译 Qt 和 Opencv 的示例，需要这样执行：

`CMAKE_PREFIX_PATH` 必须指向包含 `Qt5Config.cmake` 的目录（一般在 `.../lib/cmake/Qt5`）

`OpenCV_DIR`必须指向包含 `OpenCVConfig.cmake` 的目录（一般在 `C:\opencv\build`）

```cmake
cmake -S zxing-cpp -B zxing-cpp.build -DCMAKE_PREFIX_PATH="D:/Qt/Qt5.12.12/5.12.12/msvc2017_64/lib/cmake/Qt5" -DOpenCV_DIR="D:/opencv4-12-0/opencv/build/x64/vc16/lib" -G "Visual Studio 16 2019" -A x64
```

如果还是报：

```
INFO: Qt (Gui/Multimedia/Quick) not found, skipping Qt examples
-- Configuring done (22.1s)
CMake Warning (dev) in example/CMakeLists.txt:
  AUTOGEN: No valid Qt version found for target ZXingOpenCV.  AUTOMOC and
  AUTORCC disabled.  Consider adding:

    find_package(Qt<QTVERSION> COMPONENTS Core)

  to your CMakeLists.txt file.
This warning is for project developers.  Use -Wno-dev to suppress it.
```

按照里面的提示，在`zxing-cpp/CMakeLists.txt`中加入`find_package(Qt<QTVERSION> COMPONENTS Core)`一行，其中的换成你的版本号，这里换成`5`

```cmake
find_package(Qt5 COMPONENTS Core)
```
