## zxing-cpp

[ubuntu编译zxing-cpp踩坑](https://blog.csdn.net/lyxichigoichie/article/details/136488210)

### 不编译示例

如果只编译核心库，不编译示例，可以直接执行：

```cmake
cmake -S zxing-cpp -B zxing-cpp.build -G "Visual Studio 16 2019" -A x64
cmake --build zxing-cpp.build -j8 --config Debug
```

### 编译示例

如果要编译 `Qt` 和 `Opencv` 的示例，需要这样执行：

`CMAKE_PREFIX_PATH` 必须指向包含 `Qt5Config.cmake` 的目录（一般在 `.../lib/cmake/Qt5`）

`OpenCV_DIR`必须指向包含 `OpenCVConfig.cmake` 的目录（一般在 `C:/opencv/build`）

```cmake
cmake -S zxing-cpp -B zxing-cpp.build -DCMAKE_PREFIX_PATH="D:/Qt/Qt5.12.12/5.12.12/msvc2017_64/lib/cmake/Qt5" -DOpenCV_DIR="D:/opencv4-12-0/opencv/build/x64/vc16/lib" -G "Visual Studio 16 2019" -A x64
```

如果还是报：

```
INFO: Qt (Gui/Multimedia/Quick) not found, skipping Qt examples
-- Configuring done (22.1s)
CMake Warning (dev) in example/CMakeLists.txt:
  AUTOGEN: No valid Qt version found for target ZXingOpenCV.  AUTOMOC and
  AUTORCC disabled.  Consider adding:

    find_package(Qt<QTVERSION> COMPONENTS Core)

  to your CMakeLists.txt file.
This warning is for project developers.  Use -Wno-dev to suppress it.
```

按照里面的提示，在`zxing-cpp/CMakeLists.txt`中加入`find_package(Qt<QTVERSION> COMPONENTS Core)`一行，其中的换成你的版本号，这里换成`5`

```cmake
find_package(Qt5 COMPONENTS Core)
```

## libtiff

**libtiff 是一个开源的库**。它根据一个宽松的许可证发布，具体是 **libtiff 许可证**，该许可证与 BSD 许可证兼容。这意味着你可以在开源和闭源项目中自由使用、修改和分发它。

### 源码下载

你可以从以下几个官方或可靠的来源下载 libtiff 的源代码：

1.  **官方 GitLab 仓库（推荐）**
    这是 libtiff 项目目前最主要的开发托管地。你可以从这里获取最新的源代码。
    
    *   **主仓库地址**: [https://gitlab.com/libtiff/libtiff](https://gitlab.com/libtiff/libtiff)
    *   **下载方式**： 使用 `git` 克隆是最佳方式。
        ```bash
        git clone https://gitlab.com/libtiff/libtiff.git
        ```
    
2.  **官方发布版本（稳定版）**
    如果你希望使用经过测试的稳定版本，而不是最新的开发版，可以下载官方发布的 tar 包。
    
    *   **下载页面**: [http://download.osgeo.org/libtiff/](http://download.osgeo.org/libtiff/)
    *   在这个页面中，找到最新的版本（例如 `tiff-4.6.0.tar.gz`），点击下载。

**对于大多数用户，建议下载官方发布版本的 tar 包，因为它更稳定。**

---

### 编译指南

libtiff 使用标准的 Unix 构建系统 **Autotools**（`configure` 脚本）和 **Makefile**。在 Windows 上，你可以使用 Visual Studio 的解决方案文件或 MSYS2/MinGW 环境。

```
cmake -S . -B ../libtiff.build -G "Visual Studio 16 2019" -A x64 -DBUILD_SHARED_LIBS=ON -Dtiff-tools=OFF -DCMAKE_INSTALL_PREFIX=../libtiff.install 
cmake --build ../libtiff.build --config Debug --target install
```

### 重要的 CMake 配置选项

你可以在配置时调整这些选项：

```bash
# 禁用第三方库（如果需要），不关闭 tiff-tools 会依赖 tiffgt 导致编译出错
cmake .. -Dtiff-tools=OFF -Dtiff-tests=OFF

# 指定安装路径
cmake .. -DCMAKE_INSTALL_PREFIX=C:/libtiff

# 构建共享库（DLL）而不是静态库
cmake .. -DBUILD_SHARED_LIBS=ON

# 禁用特定功能
cmake .. -Djpeg=OFF -Dzlib=OFF

# 开启测试
cmake .. -Dtiff-tests=ON
```

### 完整的编译命令（推荐）

对于大多数用户，建议使用以下命令来避免依赖问题：

```bash
# 进入 libtiff 目录
cd libtiff

# 创建构建目录
mkdir build
cd build

# 配置（禁用工具和测试，只构建库）
cmake .. -G "Visual Studio 17 2022" -A x64 ^
  -DCMAKE_BUILD_TYPE=Release ^
  -DBUILD_SHARED_LIBS=ON ^
  -Dtiff-tools=OFF ^
  -Dtiff-tests=OFF ^
  -Dtiff-contrib=OFF ^
  -Dtiff-docs=OFF

# 编译
cmake --build . --config Release

# 安装（可选）
cmake --build . --config Release --target install
```

## simpleini

### 配置

在 `CMakeLists.txt` 中，你会看到这一行：

```CMake
add_library(${PROJECT_NAME} INTERFACE)
```

`INTERFACE` 库是 CMake 用来描述 "header-only" 库的“标准姿势”。它告诉 CMake：“我是一个库，我没有要编译的源文件，但任何链接我的目标都需要包含我的头文件。”

```cmake
# 1. 进入你下载的 SimpleIni 源码目录
cd E:\DemoProject\3rd-source

# 2. 运行 CMake 配置命令
#    -S simpleini          : 源码在当前目录
#    -B simpleini.build    : 指定构建目录为 "simpleini.build" (这个名字随意，但不能是 ".") 
#    -G "Visual Studio 16 2019" : 告诉 CMake 为 VS2019 生成项目
#    -DBUILD_TESTING=OFF : 告诉 CMake 构建测试，否则会去下载 googletest 导致配置出错
#    -DCMAKE_INSTALL_PREFIX=simpleini.install :  **这是关键！** 指定安装路径

cmake -S simpleini -B simpleini.build -G "Visual Studio 16 2019" -DBUILD_TESTING=OFF -DCMAKE_INSTALL_PREFIX=simpleini.install
```

### 安装

```CMake
cmake --install simpleini.build --config Release
```
